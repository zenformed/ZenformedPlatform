Projects are the foundation of BuildCore. Every workflow task, document, payment milestone, and accountability record ties back to a project.

This guide walks through creating your first project after you open BuildCore from your Zenformed account.

## Open the project creator

1. Sign in to **Zenformed** and launch **BuildCore** from the apps menu
2. Open the **Projects** view (your default landing page)
3. Select the **plus (+) button** in the Projects header
4. In the **New project** modal, fill in the details and select **Create project**

## Required project details

At minimum, provide:

| Field | Description |
|-------|-------------|
| **Project / customer name** | A recognizable label for your team |
| **Industry** | Choose an industry or enter a custom value |
| **Contact name** | Primary contact for the job |
| **Current stage** | Pipeline stage for the project |

> **Tip:** Use a consistent naming pattern such as `[Year] – [Customer] – [Address]` so projects are easy to find in search and reports.

## Other fields you can set now

You can also add contact email and phone, address, **Priority**, **Notes**, and **Assigned to** (including **Assign to myself**).

If your organization uses project templates, you may apply workflow task or payment blueprints to the new project draft before saving.

## After you save

Once the project is created, you'll open the project detail page. From **Project information** and the other project sections, you can:

- Review and edit project and contact details
- Add **Workflow tasks**
- Upload **Documents**
- Track **Payments** and budget entries
- Record **Accountability** items

Use **All projects** to return to the Projects list.

## Next steps

If your organization runs multi-phase jobs, read **Parent and Child Projects** to learn how to add subprojects under a parent project.
